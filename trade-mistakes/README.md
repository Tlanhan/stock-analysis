# 交易错误档案库 (Trade Mistakes Archive)

> 本目录用于记录、分析、复盘所有"偏离规则"的交易行为。
> 目的不是惩罚自己,而是把每一次错误转化为可执行的规则升级。

---

## 📁 目录结构

```
trade-mistakes/
├── README.md           # 本文件,使用说明
├── INDEX.md            # 错误索引(按时间/类型/标的检索)
├── TEMPLATE.md         # 标准记录模板,新错误从此复制
├── STATS.md            # 月度/季度统计(手工或自动生成)
└── YYYY-MM-DD-简述.md  # 每个错误一个文件
```

---

## 🏷 错误类型分类(7 类)

| 代码 | 类型 | 典型场景 |
|------|------|---------|
| **E1** | 追涨杀跌 | 突破后追,回踩割肉 |
| **E2** | 止损不严 | 设了止损线不执行,扛单 |
| **E3** | 情绪化开仓 | 没规划凭感觉买/卖 |
| **E4** | 规则外仓位 | 单笔超规则上限,或满仓全押 |
| **E5** | 错过机会 | 信号清晰但未开仓(catalyst/event 错过) |
| **E6** | 提前出场 | 浮盈被吓掉,没让利润奔跑 |
| **E7** | 加错仓 | 逆势加仓 / 浮盈不止盈反加 |

> 复合错误用 `E2+E4` 写法标记。

## 📝 记录流程

### 触发时机(任一即记录)
- 当日复盘 (`a-share-daily-review`) 发现规则偏离
- 盘中明显违规 (`a-share-intraday-review` 触发)
- 事后回看突然意识到的"昨天那笔不该开"
- 浮亏 > 5% 但**不是**因市场系统性原因

### 操作步骤
1. `cp TEMPLATE.md YYYY-MM-DD-简述.md`
2. 填完所有字段(尤其"事前规划 vs 实际操作"差异)
3. 在 `INDEX.md` 顶部追加一行
4. 如果暴露出**通用规则缺陷**→ 说"更新 skill"触发阶段 2

## 📊 文件命名规则

`YYYY-MM-DD-简述.md`

- 日期 = 错误**发生**当日(不是记录日)
- 简述 = 4-8 字中文,能一眼回忆,例:
  - `2026-06-02-COMPUTEX错过.md`
  - `2026-05-15-恒生科技追高.md`
  - `2026-04-28-情绪化清仓.md`

## 🎯 设计原则

1. **可执行**:每条错误必须产出至少一条"下次怎么避免"的具体规则
2. **可统计**:类型代码 + INDEX 让 30 天后能算出"我最常犯哪类错"
3. **可链接**:错误关联到 skill / 日历 / 催化文件,形成知识网络
4. **不伪饰**:写错误时不为自己找借口,直接写"我当时就是情绪化了"
