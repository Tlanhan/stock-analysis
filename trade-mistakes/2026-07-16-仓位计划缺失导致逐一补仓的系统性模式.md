# 2026-07-16 仓位计划缺失导致逐一补仓的系统性模式（跨5笔交易的元模式）

> **核心反思（用户原话）**："这个问题很经常出现，就是仓位没设置好，对应价位看好就买，建仓交给计划，盈亏交给市场。而不是逐一补仓。"
>
> **关键定性**：这不是某一笔交易的偶然失误，是**过去一个月内至少5次独立事件共享的同一个根因**。用户自己提炼出的原则——"建仓交给计划，盈亏交给市场"——精准点出了问题本质：正确的流程应该是**开仓前就定好价位和金额，触发条件满足就一次性执行，之后的涨跌是市场的事，不再临场加码**；而实际反复发生的是**开仓后看着价格变化，边看边补**，把本该属于"计划阶段"的决策，拖到了"执行阶段"去做，这时候决策质量必然被情绪污染。

## 1. 事前规划 vs 实际操作

### 应该的流程（用户本次提炼的原则）

1. **建仓交给计划**：开仓前先确定"在什么价位买、买多少金额"，写死下来
2. **触发即执行**：价格到了预设区间，一次性按计划金额执行，不分批试探
3. **盈亏交给市场**：执行完之后，涨跌是市场决定的事，不再因为"看着价格变化"而临场加减仓位

### 实际反复发生的模式（5次事件的共性）

### 偏离点（一句话，跨事件共性）

**我偏离了规划，因为**：开仓这个动作被拆成了多个时间点的多次小额操作，而不是一次性按预定计划执行——不管是"没有计划所以随便买"（6/15）、"计划外的标的插进来"（6/24）、"条件单没触发就手动分批追"（7/8）、"明知不该加仓仍分批加"（7/13），还是"试仓后信心不足补满"（7/16），**表现形式各不相同，但共同的根因是：仓位金额没有在开仓前被锁定为一个确定的数字，而是留了"可以随时调整"的空间，一旦留了这个空间，情绪就会利用它**。

## 4. 错误根因分析

### 三层归因

- [ ] 技术层：看错信号/读错图形/用错指标
- [x] **规则层**：现有的"加仓前三问/四问"框架（写入`a-share-trigger-check`）解决的是"要不要加仓"的判断问题，但**没有从根本上禁止"分批建仓"这种操作方式本身**——三问/四问可以在每一次分批加仓前都被拿来"过一遍"，但只要分批这个动作被允许存在，情绪就有机会在每一次分批之间找到介入的缝隙
- [x] **执行层**：即便规则本身完善，只要开仓时的金额不是一次性锁定的，后续每一次"要不要再加一点"的决策都是一次新的情绪暴露窗口

### 详细分析

过去5次事件，表面上是5个不同的失误（无计划、凑数、追高、摊平、试仓补满），但用户本次的归纳指出了一个更深层的共性：**它们都发生在"开仓"这个环节被拆分成多次操作的情况下**。如果开仓天然是一次性的（要么按计划价位一次性买入预定金额，要么不满足条件就不买），那么"越跌越买""边涨边补""试仓后补满"这些具体表现形式根本没有发生的机会——因为已经没有"第二次决策"的时间点了。

这与此前"加仓前三问/四问"框架的定位不同：三问/四问是**决策质量检查**（在动手前问自己几个问题），而用户本次提出的原则是**结构性约束**（从根本上让"分批"这个操作模式不再是默认选项）。两者应该配合使用：**如果确实需要分批（比如仓位金额较大、想分散建仓时点降低择时风险），分批的次数和每次金额也应该在第一次下单前就全部锁定，而不是"先买一点，看情况再决定要不要加"**。

## 5. 教训 → 规则升级

### 下次遇到同类场景的应对（核心新规则）

1. **开仓前必须明确回答："这笔交易的总仓位金额是多少？"**——这个数字一旦确定，除非触发了预先设定的止损/止盈条件，否则不再变动
2. **如果决定分批建仓，分批方案必须在第一笔下单前完整写出**：比如"第一笔X元@价位A，第二笔Y元@价位B（触发条件：价格达到B且满足条件C）"——不允许"先买第一笔，看情况再决定第二笔要不要买、买多少"
3. **试仓不等于留了补仓的后门**：如果选择小额试仓，试仓金额本身就应该被视为"这笔交易的完整仓位"，除非出现试仓时未曾设想过的新信号（不是"信心不足"这种主观心理变化），否则不追加
4. **"看到价格变化想加仓/减仓"的冲动出现时，先问："这个金额是不是在我最初的计划里？"**——不在计划里的操作，不管当下多有理由，都先记录下来、不立即执行，等下一次独立的复盘周期再评估是否要修改计划本身

### 需要修改的skill

- [x] **需要改skill**：`a-share-trigger-check` 的"加仓前三问/四问"框架，建议在最前面增加**"第0问"：这笔加仓，是否在开仓时就已经写入的分批计划内？如果不是，直接拒绝，不进入后续三问/四问的判断**——把"是否属于计划内操作"作为最高优先级的前置门槛，而不是和其他判断维度平级
- [ ] 待评估：是否需要在`portfolio-state.md`的"更新纪律"中增加一条——开仓时必须同时写明"本次总仓位金额"和"是否为分批计划的一部分"，作为强制字段
