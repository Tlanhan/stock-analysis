## ⏰ 数据时效性铁律（2026-06-15 新增）

### 背景
在市场分析过程中，AI 多次混淆北京时间与美东时间，导致对美股开盘时间判断错误（把21:30说成22:30）。本规则强制每次获取市场数据前先确认北京时间。

### 强制规则：获取数据前必须先确认北京时间

**每次用户要求获取市场数据（A股/美股/港股/期货/外汇）时，必须先执行：**

```python
from datetime import datetime, timezone, timedelta
bj = timezone(timedelta(hours=8))
print(datetime.now(bj).strftime('%Y-%m-%d %H:%M:%S %A'))
```

**然后基于北京时间判断各市场状态：**

| 市场 | 交易时段（北京时间） |
|------|-------------------|
| A股 | 周一-周五 9:30-11:30, 13:00-15:00 |
| 港股 | 周一-周五 9:30-12:00, 13:00-16:00 |
| 美股（夏令时3月中-11月初） | 周一-周五 **21:30**-次日4:00 |
| 美股（冬令时11月初-3月中） | 周一-周五 **22:30**-次日5:00 |
| 日股 | 周一-周五 8:00-14:00 |

**数据获取后必须标注**："数据获取时间：北京时间 YYYY-MM-DD HH:MM"

### 反铁律

- ❌ 不确认当前时间就获取数据并分析
- ❌ 混淆北京时间与美东时间
- ❌ 用过期缓存数据当实时数据
- ❌ 不标注数据获取的具体时间
- ❌ 假设系统时区正确（系统时区可能配置错误，必须用Python UTC+8手动转换）

## 🔧 工具使用注意事项

### ETF数据获取
- kimi-datasource（query_stock）**不支持ETF查询**，对ETF返回"No realtime data available"
- 获取ETF实时数据应使用 a-stock-analysis skill（新浪财经接口）
- 159558等深交所新ETF可能新浪接口也无法获取，需用东方财富/雪球补充

### 时间转换
- 系统时区可能配置错误，所有时间判断必须通过Python UTC+8手动转换
- 不要依赖 `date` 命令的输出（可能使用错误时区）
- 美股夏令时：3月第二个周日 → 11月第一个周日
