# 规则速查表（从 AGENTS.md + CLAUDE.md 压缩）

> **用途**：高压时刻（盘中犹豫、用户说"蒙了"）快速查表，不用翻 700 行规则文件。
> **权威源**：完整规则仍在 `AGENTS.md`（做什么）和 `CLAUDE.md`（怎么做）。本文件是速查索引。
> **设计原则**：硬规则 5 条 + 状态机 4 个 + 查表指引。

---

## 🔒 五条硬规则（不可违反）

| # | 规则 | 一句话 | 违反后果 |
|:-:|------|--------|---------|
| 1 | **校准时间** | 会话第一件事：Python 确认北京时间 + 周/月范围 | 用错日期=整篇报告作废 |
| 2 | **先真值后出卡** | 用户说"想交易"→ 先拉 qt_quote + em_data 陈述事实 → 再出条件卡 | 空对空决策=赌博 |
| 3 | **消息三层检验** | 双日期标注 + 旧闻识别 + 出货信号检查 | 把旧闻当利好=高位接盘 |
| 4 | **不推不push** | 用户不说"推送"→ 禁止 git push | 污染远程仓库 |
| 5 | **单标的专注** | 同一交易日只允许 1 个主线标的 | 凑数交易=分散注意力=主线卖飞 |

## 🔄 会话状态机（4 个状态，按需切换）

```
[空闲] → 用户说"复盘/分析/行情" → [讨论模式]
   ↓ 用户说"更新skill"
[讨论] → 用户说"写卡/想买" → [决策模式]（强制调用 trigger-check）
   ↓ 用户说"更新skill"
[讨论/决策] → Edit skills → [更新模式]
   ↓ 用户说"推送"
[更新] → cp + commit + push → [空闲]
```

**每个状态只加载该状态需要的规则。**

## 📋 查表指引（按场景）

### 场景 A：盘前规划（9:00-9:15）

```
1. Read portfolio-state.md（当前持仓）
2. Read market-regime.md（环境参数：单笔上限/止损幅度）
3. Read sentiment-calendar-YYYY-MM.md（今日是否强扰动）
4. Read oversea-catalyst-YYYY-WNN.md（海外催化）
5. qt_quote 拉昨收+竞价
6. em_data 拉板块/主力资金
7. trigger-check 模式A 写卡（强制含：主盯标的1个/挂单标的/加仓上限）
```

### 场景 B：盘中犹豫（"蒙了/想卖/想追"）

```
1. 先拉真值（qt_quote 四指数+用户标的 + em_data 板块/主力）
2. 摆事实不解释（"深指+1.2% 沪指-0.3%，半导体主力+28亿，你的159558现3.51"）
3. 过 pre-trade-checklist.md（16项，30秒）
4. 归到操作（GO/NO-GO）
```

### 场景 C：收盘复盘（15:00+）

```
1. a-share-daily-review → 10章报告
2. 条件卡闭环验证（condition-card-validator.md）
3. 更新 trade-log-YYYY-MM.md
4. 更新 portfolio-state.md（持仓/资金）
5. 有失误 → trigger-check 模式C 归因
```

### 场景 D：想买某个标的

```
1. 触发 trigger-check 模式A/B
2. 从 market-regime.md 取参数（单笔上限/止损/追高容忍度）
3. 过 pre-trade-checklist.md
4. 输出条件交易卡（IF-THEN）
```

## 🚫 速查禁止项

| 禁止 | 违反时的正确做法 |
|------|----------------|
| 用户说"蒙了"时直接问"你觉得呢" | 先拉真值，再引导 |
| 用分析替代执行框架 | 先出条件卡，再补分析 |
| 追高（5日线乖离 > 5%）| 等回踩，不追 |
| 卖飞后48小时内以更高价追回 | 强制冷却期 |
| 凑数交易（非计划内标的填仓位）| 宁可仓位不满 |
| 同时盯2+标的（高波动环境）| 主盯1个，其余挂单 |
| 尾盘手动下单（14:50后）| 只允许条件单 |
| 不校准时间就拉数据 | Python UTC+8 先 |
| API 不可用直接用新闻冒充价格 | 标"无数据"，不编造 |
